โดยเน้นการออกแบบระบบให้มีมาตรฐาน รองรับการขยายระบบในอนาคต และสามารถนำไปสาธิตการทำงานได้จริง

- พัฒนาระบบจัดการกิจกรรมของสถานศึกษา

## บทบาทและเป้าหมาย

คุณคือนักพัฒนาเว็บที่ต้องสร้างเว็บแอป **"Rmutr School" (School Activity Management System)** สำหรับบริหารจัดการกิจกรรมของสถานศึกษา ให้นักเรียนลงทะเบียนเข้าร่วมกิจกรรม เช็กชื่อด้วย QR Code สะสมคะแนน/ชั่วโมงกิจกรรมโดยอัตโนมัติ และรับใบประกาศนียบัตรเมื่อจบกิจกรรม โดยมีหน้า Dashboard แยกตามสิทธิ์ผู้ใช้ 3 ระดับ (Admin / Teacher / Student) และมีระบบรายงาน/Export ให้ผู้บริหารและครูใช้ติดตามผล

---

## 1. Tech Stack และโครงสร้างไฟล์ (บังคับ)

- **ห้ามใช้ framework** (ไม่ใช้ React/Vue/Next) — ใช้ **Vanilla HTML + CSS + JavaScript (ES6+)** เท่านั้น
- **แยกไฟล์เด็ดขาด** ห้ามรวม CSS/JS ไว้ในไฟล์เดียวกับ HTML (ห้ามใช้ `<style>` หรือ `<script>` inline ยกเว้นการ import ไฟล์)
- ฐานข้อมูลและ storage ใช้ **Supabase** (Postgres + Supabase Auth สำหรับ login ทุก role + Row Level Security (RLS) บังคับสิทธิ์ตาม role)
- ใช้ **Fetch API + Async/Await** ในการเรียก Supabase ทั้งหมด
- โครงสร้างไฟล์ที่ต้องสร้าง:

> **หมายเหตุไอคอน:** ไม่ต้องดาวน์โหลดไอคอนมาเก็บไว้ในเครื่อง ให้ **ดึงไอคอนจาก Flaticon ผ่านลิงก์ URL โดยตรง** (เช่น `<img src="https://cdn-icons-png.flaticon.com/...">` หรือลิงก์ embed ที่ Flaticon ให้มา) แล้วอ้างอิง URL นั้นไว้ในไฟล์ JS/HTML ที่เกี่ยวข้องแทน

## 3. Role & สิทธิ์การใช้งาน (RBAC)

| Role | สิทธิ์ |
|---|---|
| **Admin** | สิทธิ์ทั้งหมด: จัดการผู้ใช้, สิทธิ์, ประเภทกิจกรรม, ปีการศึกษา, ภาควิชา, ห้องเรียน, ข้อมูลทั้งหมด, ดูรายงานทั้งหมด, ดู System Logs |
| **Teacher** (รวมบทบาท "เจ้าหน้าที่" เป็น role เดียว) | สร้าง/แก้ไข/ลบกิจกรรม (เฉพาะที่ตัวเองรับผิดชอบ), **เห็นกิจกรรมทั้งหมดในระบบแบบดูอย่างเดียว**, จัดการนักเรียนในกิจกรรม, เช็กชื่อ, ประเมิน/ให้คะแนน, อนุมัติการลงทะเบียน |
| **Student** | ลงทะเบียนกิจกรรม, ยกเลิกลงทะเบียนก่อนวันกิจกรรมเริ่ม, ดูสถานะการลงทะเบียน, ดูประวัติย้อนหลัง, ดูคะแนน/ชั่วโมงสะสม, ดาวน์โหลดใบประกาศนียบัตร |

**กติกาการมองเห็นของ Teacher (สำคัญ ต้องบังคับผ่าน Supabase RLS):**
- Policy `SELECT` บนตาราง `activities`: ครูทุกคนอ่านได้ทุกแถว
- Policy `UPDATE`/`DELETE` บนตาราง `activities`: อนุญาตเฉพาะเมื่อ `auth.uid()` มีอยู่ใน `activity_teachers` ของ activity นั้น

## 4. ฟีเจอร์แต่ละหน้า

### 4.1 หน้า Login (index.html)
- Login ด้วย Supabase Auth (email/password)
- Reset Password / เปลี่ยนรหัสผ่าน
- หลัง login แล้ว redirect ไปหน้า dashboard ตาม role อัตโนมัติ

### 4.2 Admin
- **จัดการผู้ใช้งาน:** เพิ่ม/แก้ไข/ลบผู้ใช้, กำหนด role
- **จัดการประเภทกิจกรรม / ปีการศึกษา / ภาควิชา / ห้องเรียน:** CRUD ครบทุกตาราง
- **System Logs:** ดูประวัติ Login, Logout, เพิ่ม/แก้ไข/ลบข้อมูล, เช็กชื่อ, ให้คะแนน (read-only, filter ตามวันที่/ผู้ใช้/ประเภท action)

### 4.3 Teacher
- **จัดการกิจกรรม:** เพิ่ม/แก้ไข/ลบ (เฉพาะกิจกรรมของตัวเอง), ระบุประเภท, วันที่, เวลา, สถานที่, จำนวนที่นั่งสูงสุด (`max_participants`), เปิด/ปิดการอนุมัติล่วงหน้า (`requires_approval`)
- **กำหนดครูผู้รับผิดชอบ:** เพิ่ม/ลบครูในกิจกรรม (many-to-many ผ่าน `activity_teachers`) พร้อมระบุบทบาท (หัวหน้า/ผู้ช่วย)
- **ดูรายชื่อนักเรียนที่ลงทะเบียน:** อนุมัติ/ปฏิเสธการลงทะเบียน (สำหรับกิจกรรมที่ `requires_approval = true`)
- **เช็กชื่อ/ให้คะแนน:** ดูรายชื่อผู้เข้าร่วม, กรอกคะแนนจิตพิสัยและชั่วโมงกิจกรรมต่อคน, mark สถานะเป็น completed
- **QR Code:** generate QR code 1 ชุดต่อกิจกรรม (ผูกกับ `activity_id`) และแสดง/พิมพ์ให้นักเรียน scan ที่หน้างาน

### 4.4 Student
- **รายการกิจกรรม:** ดูกิจกรรมที่เปิดลงทะเบียน, ค้นหา (ชื่อ/รหัสกิจกรรม), filter (ปีการศึกษา, ประเภท, วันที่, สถานะ, ห้องเรียน)
- **ลงทะเบียน:** กดลงทะเบียน → ระบบเช็คที่นั่งคงเหลือ (`max_participants`) และเช็คว่าเคยลงทะเบียน activity นี้แล้วหรือยัง (ห้ามซ้ำ) → ถ้ากิจกรรมต้องอนุมัติ สถานะเป็น `pending_approval`, ถ้าไม่ต้อง สถานะเป็น `registered` ทันที
- **ยกเลิกลงทะเบียน:** ทำได้ก่อนวันกิจกรรมเริ่มเท่านั้น
- **เช็กชื่อด้วย QR:** เปิดกล้องมือถือ scan QR ของกิจกรรม → ระบบตรวจสอบว่ามี registration ที่ approved อยู่ → บันทึก `check_in_time` และ mark สถานะเป็น `completed` ทันที (ไม่มี check-out แยก)
- **ดูคะแนน/ชั่วโมงสะสม:** สรุปรวมจากทุกกิจกรรมที่ completed
- **ใบประกาศนียบัตร:** ระบบออกให้อัตโนมัติทันทีที่สถานะเป็น completed (ไม่มีเกณฑ์คะแนน/ชั่วโมงขั้นต่ำ) — ดาวน์โหลดเป็น PDF ได้จากหน้านี้

### 4.5 รายงาน (ทุก role เข้าถึงได้ตามสิทธิ์)
- **Dashboard:** จำนวนกิจกรรมทั้งหมด, กิจกรรมวันนี้, นักเรียนทั้งหมด, นักเรียนที่เข้าร่วม, ชั่วโมงกิจกรรมรวม, คะแนนรวม, กราฟรายเดือน, กราฟประเภทกิจกรรม, กิจกรรมล่าสุด
- **รายงานกิจกรรม / นักเรียน / คะแนน / ชั่วโมงกิจกรรม:** ตาราง + filter
- **Export:** PDF, Excel, CSV ครบทั้ง 3 แบบ (ใช้ jsPDF และ SheetJS)

## 5. โครงสร้างฐานข้อมูล Supabase (SQL Schema ที่ต้องสร้าง)

```sql
-- ใช้ Supabase Auth (auth.users) เป็นฐาน แล้วเก็บ profile/role เพิ่มเติมที่นี่
create table profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  full_name text not null,
  role text not null check (role in ('admin', 'teacher', 'student')),
  created_at timestamptz default now()
);

-- ภาควิชา/แผนก
create table departments (
  id uuid primary key default gen_random_uuid(),
  name text not null
);

-- ปีการศึกษา
create table academic_years (
  id uuid primary key default gen_random_uuid(),
  year_name text not null,          -- เช่น '2569'
  start_date date not null,
  end_date date not null,
  is_active boolean default false
);

-- ห้องเรียน
create table classes (
  id uuid primary key default gen_random_uuid(),
  class_name text not null,         -- เช่น 'ม.6/1'
  department_id uuid references departments(id),
  academic_year_id uuid references academic_years(id)
);

-- นักเรียน
create table students (
  id uuid primary key references profiles(id) on delete cascade,
  student_code text unique not null,
  class_id uuid references classes(id)
);

-- ครู
create table teachers (
  id uuid primary key references profiles(id) on delete cascade,
  teacher_code text unique not null,
  department_id uuid references departments(id)
);

-- ประเภทกิจกรรม
create table activity_types (
  id uuid primary key default gen_random_uuid(),
  name text not null               -- เข้าค่าย, กีฬาสี, วันวิชาการ, จิตอาสา, ชุมนุม, ค่ายอบรม, อื่นๆ
);

-- กิจกรรม
create table activities (
  id uuid primary key default gen_random_uuid(),
  activity_code text unique not null,
  name text not null,
  activity_type_id uuid references activity_types(id),
  academic_year_id uuid references academic_years(id),
  description text,
  location text,
  start_datetime timestamptz not null,
  end_datetime timestamptz not null,
  max_participants int,
  requires_approval boolean default false,
  qr_code text unique,
  status text default 'draft' check (status in
    ('draft','published','registration_open','registration_closed','ongoing','completed','cancelled')),
  created_by uuid references profiles(id),
  created_at timestamptz default now()
);

-- ครูผู้รับผิดชอบกิจกรรม (many-to-many)
create table activity_teachers (
  id uuid primary key default gen_random_uuid(),
  activity_id uuid references activities(id) on delete cascade,
  teacher_id uuid references teachers(id) on delete cascade,
  role_in_activity text default 'ผู้ช่วย',   -- 'หัวหน้า' | 'ผู้ช่วย'
  unique (activity_id, teacher_id)
);

-- การลงทะเบียน
create table registrations (
  id uuid primary key default gen_random_uuid(),
  student_id uuid references students(id) on delete cascade,
  activity_id uuid references activities(id) on delete cascade,
  status text default 'registered' check (status in
    ('pending_approval','approved','rejected','registered','cancelled')),
  registered_at timestamptz default now(),
  unique (student_id, activity_id)   -- ห้ามลงทะเบียนกิจกรรมเดียวกันซ้ำ
);

-- การเข้าร่วม + คะแนน + ชั่วโมง (รวมตารางเดียว)
create table attendance (
  id uuid primary key default gen_random_uuid(),
  student_id uuid references students(id) on delete cascade,
  activity_id uuid references activities(id) on delete cascade,
  check_in_time timestamptz,
  status text default 'registered' check (status in
    ('registered','present','absent','late','completed')),
  score numeric,
  hours_earned numeric,
  graded_by uuid references teachers(id),
  updated_at timestamptz default now(),
  unique (student_id, activity_id)
);

-- ใบประกาศนียบัตร
create table certificates (
  id uuid primary key default gen_random_uuid(),
  student_id uuid references students(id) on delete cascade,
  activity_id uuid references activities(id) on delete cascade,
  certificate_no text unique not null,
  issue_date date default current_date,
  file_url text,
  unique (student_id, activity_id)
);

-- System Logs
create table logs (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles(id),
  action text not null,             -- 'login' | 'logout' | 'create' | 'update' | 'delete' | 'check_in' | 'grade'
  table_name text,
  record_id uuid,
  detail jsonb,
  created_at timestamptz default now()
);

-- ตั้งค่าระบบทั่วไป
create table settings (
  id int primary key default 1,
  school_name text default 'Rmutr School',
  logo_url text
);
```

**Row Level Security (RLS) ที่ต้อง enable และเขียน policy:**
- ทุกตาราง: enable RLS
- `activities`: SELECT เปิดให้ authenticated ทุก role, UPDATE/DELETE เฉพาะ admin หรือครูที่อยู่ใน `activity_teachers` ของ activity นั้น
- `registrations`, `attendance`, `certificates`: นักเรียนเห็น/แก้ไขได้เฉพาะแถวของตัวเอง (`student_id = auth.uid()`), ครู/admin เห็นได้ทุกแถวที่เกี่ยวข้องกับกิจกรรมของตน
- `logs`: อ่านได้เฉพาะ admin

**Storage buckets ที่ต้องสร้างใน Supabase:**
- `certificates` — เก็บไฟล์ PDF ใบประกาศนียบัตรที่ generate แล้ว
- `assets` — เก็บโลโก้/รูปประกอบอื่นๆ

## 6. Business Logic สำคัญที่ต้อง Implement

- **กันลงทะเบียนซ้ำ:** unique constraint `(student_id, activity_id)` ที่ตาราง `registrations` + เช็คฝั่ง client ก่อนแสดง error แบบ pop-up
- **จำกัดที่นั่ง + ปิดรับอัตโนมัติเมื่อเต็ม:** เขียนเป็น Postgres function/trigger นับจำนวน `registrations` ที่ status `registered`/`approved` เทียบกับ `max_participants` ก่อนอนุญาต insert ใหม่ (ป้องกัน race condition เวลาที่นั่งสุดท้ายมีคนสมัครพร้อมกัน)
- **QR Code เช็กชื่อ:** ผูกกับ `activity_id` (โค้ดเดียวกันทุกคนใช้ scan ร่วมกัน), นักเรียน scan เองด้วยมือถือ, scan ครั้งเดียว = อัปเดต `attendance.status = 'completed'` ทันที (ไม่มี check-out แยก)
- **ออก Certificate อัตโนมัติ:** trigger เมื่อ `attendance.status` เปลี่ยนเป็น `completed` ให้ insert แถวใหม่ใน `certificates` พร้อม generate `certificate_no` (running number ต่อปีการศึกษา) และสร้างไฟล์ PDF เก็บใน storage bucket `certificates`

## 8. ข้อควรระวัง/กติกาการเขียนโค้ด

- ห้ามใช้ `alert()`, `confirm()`, `prompt()` ของ browser — ใช้ `popup.js` เท่านั้น
- ห้ามฝัง API key ของ Supabase แบบ hardcode ในไฟล์ที่ commit ขึ้น public repo — ใช้ตัวแปร config แยกในไฟล์เดียว (`supabase-client.js`) เพื่อสลับ environment ตอน deploy จริงได้ง่าย
- ไอคอนทั้งหมดดึงผ่านลิงก์ URL จาก Flaticon โดยตรง ไม่ต้องดาวน์โหลดไฟล์มาเก็บไว้ในโปรเจกต์
- ทุกฟีเจอร์ต้องทดสอบบน 3 ขนาดจอ: Desktop (>1200px), Tablet (768–1024px), Mobile (<480px)
- โค้ดทุกไฟล์ใน `/js/` ต้องแยกหน้าที่ชัดเจนตามชื่อไฟล์ ห้ามเขียนทุกอย่างไว้ไฟล์เดียว
- ห้ามเขียน CSS/JS รวมใน HTML (ยกเว้นกรณีจำเป็นจริงๆ)
- ใช้ comment ภาษาไทยหรืออังกฤษกำกับฟังก์ชันหลักเพื่อให้แก้ไขง่ายในอนาคต
- Sensitive logic (การนับที่นั่งคงเหลือ, การเช็คสิทธิ์แก้ไขกิจกรรม) ต้องพึ่ง Supabase RLS/Function เป็นหลัก ไม่ใช่แค่ซ่อนปุ่มฝั่ง client
